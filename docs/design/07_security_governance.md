# 07. Security & Governance

Sentinel is designed with a "Security-First" philosophy, specifically tailored for enterprise fraud investigation environments where data sensitivity and auditability are paramount.

## 🛡️ 1. Authentication & RBAC

Sentinel uses **Role-Based Access Control (RBAC)** to enforce the principle of least privilege:

- **Admin**: Full system access, including user management, API key generation, and database maintenance.
- **Investigator**: Full access to RAG, Trends, and ML scoring/training. Cannot modify system users.
- **Guest/Viewer**: Read-only access to dashboards and generated reports.

### Authentication Flow:
1. **User Login**: Streamlit calls `/auth/token` with credentials.
2. **JWT Issuance**: Backend validates against the `app_users` table (bcrypt hashed passwords) and returns a signed HS256 JWT.
3. **Session Persistence**: The token is stored in Streamlit's `session_state` and passed in subsequent API headers.

## 📜 2. Forensic Audit Logging

Every critical action within the workbench is logged to ensure a clear "Chain of Evidence":

- **Action Logging**: All queries, model training runs, and data uploads are timestamped with the associated User ID.
- **Agentic Audit**: The RAG pipeline logs the exact SQL and Cypher queries generated by agents, allowing investigators to verify the logic behind AI-generated insights.
- **Model Registry**: Versions of models are immutable once deployed to ensure that past fraud scores can be reproduced exactly as they were at the time of scoring.

## 🔏 3. Data Safety & Privacy

- **Local-First Inference**: By integrating with **Ollama** and **MLX**, sensitive transaction data can be processed on-premises without leaving the corporate network.
- **SQL Sanitization**: Before executing agent-generated SQL, the system performs validation checks to prevent destructive operations (e.g., `DROP`, `DELETE`).
- **Input Filtering**: Multimodal inputs (images/videos) are processed in isolated buffers before being converted into vector embeddings.

## ⚖️ 4. AI Ethics & Governance

Sentinel prevents AI hallucinations through **Grounding**:
- The **Validator Agent** cross-checks LLM synthesis against the raw query results.
- **Confidence Scoring**: Every AI insight is accompanied by a confidence metric (e.g., "78% Match with Knowledge Base").
- **Human-in-the-Loop**: No model is "Auto-Deploys" without an Investigator reviewing the SHAP/Metrics report first.
